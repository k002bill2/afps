# 앱 종합 검증 워크플로우

이 커맨드는 Boris Cherny가 강조하는 **검증 피드백 루프**를 구현합니다.
모든 코드 변경 후 이 커맨드를 실행하여 품질을 보장하세요.

## 검증 단계

### 1. TypeScript 타입 체크
```bash
npm run type-check
```
**통과 기준**: 타입 에러 0개

### 2. ESLint 린트 검사
```bash
npm run lint
```
**통과 기준**: 린트 에러 0개 (경고는 허용)

### 3. Jest 테스트 실행
```bash
npm test -- --coverage --coverageReporters="text-summary"
```
**통과 기준**:
- 모든 테스트 통과
- Statements: 75% 이상
- Functions: 70% 이상
- Branches: 60% 이상

### 4. 개발 빌드 검증
```bash
npm run build:development
```
**통과 기준**: 빌드 성공

## 결과 리포트 형식

## 실패 시 조치

### TypeScript 에러
1. 에러 위치 확인
2. 타입 정의 수정 또는 추가
3. `any` 대신 `unknown` 사용

### ESLint 에러
1. 자동 수정 시도: `npm run lint -- --fix`
2. 수동 수정 필요 시 규칙 확인

### 테스트 실패
1. 실패한 테스트 확인
2. 예상값 vs 실제값 비교
3. 코드 또는 테스트 수정

### 빌드 실패
1. 빌드 로그 확인
2. 의존성 문제 확인: `npm install`
3. 캐시 정리: `npm run clean`

## 자동화 권장

### 코드 변경 후
모든 파일 편집 후 자동으로 이 검증을 실행하세요.

### PR 생성 전
`/commit-push-pr` 실행 전에 반드시 `/verify-app`을 실행하세요.

### CI/CD 연동
GitHub Actions에서 동일한 검증을 수행하도록 설정하세요.
